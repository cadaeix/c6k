# docs/ — your build's own notes

This is where *your* architecture and philosophy notes live: the self-contained "what is this
character, how does it work, and why is it shaped this way" for your specific build.

Two docs worth keeping separate (the kit's own `../../docs/` does the same split):

- **architecture** — the machine: how your character's files, hooks, memory, corpus, and any
  heartbeat actually fit together. The doc you'd hand someone who needs the *how*.
- **philosophy** — the *why*: your stances, and how each one became an architecture decision (the
  belief → failure-state → architecture chain from the kit's `philosophy.md`).

Keeping the reasoning here — in the workshop, not in the character's own files — means you can
read and revise it without waking the character.
